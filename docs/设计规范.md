# 3:4 图文卡片 · 设计规范

> 基于 frontend-slides 项目 Paper & Ink 预设改造，经多轮迭代沉淀。
> 设计画布 1080×1440 (3:4)，单 HTML 文件，零外部依赖（字体除外）。

---

## 一、画布与布局

| 属性 | 值 | 说明 |
|---|---|---|
| 画布尺寸 | **1080 × 1440 px** | 3:4 竖版比例 |
| 卡片边距 `--pad` | **52 px** | 上下左右统一，文字与图片均不超出此边界 |
| 缩放方式 | JS 动态计算 | `scale = min(视口宽/1080, 视口高/1440) × 0.94` |
| 定位方式 | `transform: translate() scale()` | 从左上角 (0,0) 计算偏移，居中显示 |
| 视口容器 | `position:fixed; inset:0; overflow:hidden` | 无 flex 居中，避免与 transform 冲突 |

### 可用空间计算

```
内容宽度 = 1080 - 52×2 = 976 px
内容高度 = 1440 - 52×2 = 1336 px
每行高度 = 40 × 1.6 = 64 px（正文）
每行字符数 ≈ 976 ÷ 40 ≈ 24 字（汉字）
每页最大行数 ≈ 1336 ÷ 64 ≈ 20 行
```

## 三、内容规范

### 3.1 原文原则

- **不得修改原文内容** — 必须使用 markdown 源文件中的逐字原文
- 段落标题（如 `**场景一：硅谷的繁荣叙事。**`）以加粗 `<p>` 嵌入正文流，**不独占卡片**
- 大章节标题（如 `## 一、两个极端`）同样以加粗 `<p>` 嵌入，不用 `<h2>` 避免换页

### 3.2 卡片切分逻辑

```
判断当前页剩余空间能否放下下一段文字：
  能 → 继续追加
  否 → 新建卡片

判断当前页能否放下图片 + 图名：
  能 → 本页末尾插入图片
  否 → 本页只放文字，下一页以图片开头
       → 图片之后继续写文章
```

### 3.3 文字容器

- 纯文字卡片使用 `.txt`（`flex:1; justify-content:center`）— 内容垂直居中，空白均匀分布
- 图文混合卡片：文字用 `.txt`，图片用 `.imgbx`（`flex:1`），两者平分剩余空间

### 3.4 段落标题不换页

所有 `##` 大标题和 `**` 加粗小标题都**不另起卡片**，直接跟在上一段后面。只有文字自然撑满 1336px 时才换页。

## 四、图片规范

| 属性 | 值 |
|---|---|
| 宽度 | **卡片宽度的 90%**（`max-width:90%`） |
| 高度约束 | **`max-height:100%`**（容器剩余空间） |
| 缩放方式 | `object-fit:contain` — 完整显示不变形 |
| 图名字号 | **22px**，灰色斜体 |
| 图名位置 | 图片下方居中 |

### 4.1 图片放置逻辑

```
if 文字后剩余空间 ≥ 图片高度 + 图名高度:
    图片放在本页末尾
else:
    图片放在下一页顶部（作为该页首个元素）
    文字在图片之后继续
```

图片上下边缘与 `--pad` 边距对齐。

### 4.2 图片在页首时

```css
.imgbx{flex:0 0 auto;justify-content:flex-start;min-height:auto}
img{max-height:520px}
```

固定图片高度 520px，避免撑满整页。

## 六、字体修正规则

所有**同时包含中文和数字/拉丁字符**的元素必须使用 `var(--fb)`（Source Serif 4），避免显示字体 `var(--fd)`（Cormorant Garamond）的拉丁数字与 fallback 中文字号不匹配。

**需要修正的典型场景：**
- 场景标题："每年 **2%**"
- 大数字："**1** 亿倍"
- 年份标签："**2000** 年"
- 百分比："**3%** 保留给人类"
- 含数字的引用块
- 含拉丁字符的标题

**不需要修正的场景（纯文本）：**
- 纯中文标题
- 纯数字装饰元素（页码、背景编号）
- 纯英文元素（导航按钮）
